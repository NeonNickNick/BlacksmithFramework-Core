# 判定实现
[返回](./引言.md)

介绍动态注册系统的设计意图和适用场景——通过 `RegistCallbackOnJudge` 向判定管线注册回调。完整执行链路（每个阶段在何时触发、由哪段代码驱动）见 [判定流程](../判定流程.md)。

> **HPB 关键变更**：动态注册从委托模式改为 `AnalyzerKey` + `AnalyzerRegistry.JudgeCallback` 调度。`RuleUnit` 不再携带 `Action<Community, Community> Rule`，而是 `AnalyzerKey` 字符串 + `IsPlayer` 标志。

## 动态注册解决什么问题

默认的阶段链对所有技能一视同仁——双方攻击对消、效果按类型触发、按序结算。动态注册允许单个技能在特定阶段插入额外逻辑：改写对消结果、延迟对手攻击、触发自动反击、挂载跨回合状态检查和清理回调。

## StageRuleContainer 执行模型

每个阶段持有一个 `StageRuleContainer`，内含 `_baseRule`（默认规则，`Action<Community, Community>`）、`_overrideRules`（覆盖规则列表）、`_modifiersBefore` / `_modifiersAfter`（前后修饰器列表）。所有覆盖和修饰器条目以 `RuleUnit` 形式存储：

```csharp
public class RuleUnit : IAnalyzableData
{
    public required string AnalyzerKey { get; init; }  // 指向 AnalyzerRegistry.JudgeCallback
    public required ClapRoundClock Clock { get; init; }
    public required bool IsPlayer { get; init; }
}
```

执行顺序固定为 **Before Modifiers（仅 `IsRinging` 的） → 核心（取最后一个 `IsRinging` 的 Override，否则 Base） → After Modifiers（仅 `IsRinging` 的）**。执行完毕后自动 `Update()`：移除所有 `IsDead` 规则，对剩余规则调用 `Clock.RoundPass()` 推进回合。

执行时通过 `AnalyzerRegistry.JudgeCallback` 调度：

```csharp
// 核心执行逻辑（简化）
// Before modifiers
foreach (var rule in _modifiersBefore)
{
    if (rule.Clock.IsRinging)
    {
        if (rule.IsPlayer)
            AnalyzerRegistry.JudgeCallback.Get(rule.AnalyzerKey)(player, enemy);
        else
            AnalyzerRegistry.JudgeCallback.Get(rule.AnalyzerKey)(enemy, player);
    }
}
// 核心规则（类似逻辑）
// After modifiers（类似逻辑）
```

## 模板专门化

`ICallbackOnJudge` 通过 `IsPlayer` 标志固化视角。注册时 `AddJudgeRule(source, callbacks)` 将 `source` 的玩家身份记录为 `IsPlayer`。执行时：

- `IsPlayer == true` → 调用 `AnalyzerRegistry.JudgeCallback.Get(key)(player, enemy)`
- `IsPlayer == false` → 调用 `AnalyzerRegistry.JudgeCallback.Get(key)(enemy, player)`

这使得回调方法始终接收 `(player, enemy)` 二参数，框架自动处理双方身份的对应关系。

## 适用场景

**适合用动态注册**：
- 改写特定阶段的行为（反射、转移、攻击翻倍）
- 下回合触发（延迟检查、清理、重置）
- 自动反击（在对手攻击前挂载额外攻击）
- 延迟保护（将即时攻击/效果推迟到下一回合）
- 与默认规则耦合的效果（在对消前后拦截）
- 纯回合事件回调（状态重置、标记清理、分身死亡检测）

**不适合用动态注册**：
- 单纯的攻击/防御/资源变化——用普通 DSL 语句
- 一次性效果（不依赖阶段顺序）——用 `WriteCompileTime` 或 `WriteEffect`
