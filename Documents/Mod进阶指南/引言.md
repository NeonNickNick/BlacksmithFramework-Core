## 进阶能力依赖

基础指南足够写出普通攻击、防御、资源变化、简单被动技能。以下能力需要进阶机制：

- 临时改写某个判定阶段
- "下一回合"开始生效的技能
- 在特定阶段插入额外规则
- 转移、反击、延迟保护、阶段性强制处理
- 自定义效果（通过 `[IsAnalyzer]` 注册）
- 自定义防御（通过 `DefenseEntity` + `[IsAnalyzer(AnalyzerType.Defense)]`）

这些主要依赖：

- `SourceFile`（`IDSLSourceFile`）/ `WriteFree` 与 Mark 系统
- `ISudoOperations` 越权操作
- `ICallbackOnJudge` / `ModifierCallback` / `OverrideCallback` / `JudgeRuleManager`（见[判定实现](./判定实现.md)）
- `AnalyzerRegistry` + `[IsAnalyzer]`

## 核心结论

1. DSL 不直接结算，而是构造 `SourceFile`
2. `SourceFile.Compile(judgeRuleManager)` 产出 `Intent`
3. `RegistCallbackOnJudge(...)` 在编译阶段将 `ICallbackOnJudge` 注册到裁判系统。回调使用 `AnalyzerKey` + `IsPlayer` → `AnalyzerRegistry.JudgeCallback` 调度
4. `JudgeRuleManager` 按 9 个 `JudgeStage` 依次运行基础规则、覆盖规则和前后修饰器
5. Mark 系统（`AddMark`/`TakeMark`/`CountMark` + `out Lazy<T>`）提供跨回合状态管理，注意数据冒险约束
