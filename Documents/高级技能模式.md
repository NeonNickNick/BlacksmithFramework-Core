# 高级技能模式

本文档讲解需要跨回合状态、动态注册、自定义效果/防御、条件性执行或越权操作的技巧性技能实现。

---

## 1. Mark 系统深度用法

Mark 是存储于 `Effect` 组件的无限期 `EffectEntity`（`IsMark = true`, `Type = Default`, `isInfinite: true`），按 `AnalyzerKey` 标识。它替代了传统的状态变量，是跨回合状态管理的核心机制。

### 1.1 基本操作

#### DSL 链式方法（声明时，执行时生效）

```csharp
.AddMark("myMark")                                    // 写入一个标记
.TakeMark("myMark", out Lazy<int> layerNum)            // 取出所有同名标记并返回数量
.CountMark("myMark", out Lazy<int> layerNum)           // 只计数不移除
.TakeMark(markNameSet, out Lazy<IReadOnlyDictionary<string, int>> layerNums)  // 批量
```

#### Body 扩展方法（立即执行）

用于 Analyzer 或 JudgeCallback 内部（已在执行阶段）：

```csharp
body.AddMark("myMark")                        // 立即写入
int count = body.CountMark("myMark")          // 立即计数
int taken = body.TakeMark("myMark")           // 立即取出
```

### 1.2 典型模式：叠层增伤（Cannon 三连击）

```csharp
// 每次使用三连击，取出之前的叠层 → 攻击力 + 叠层数 → 重新写入叠层
private static IDSLSourceFile TripleStrike(ISkillContext sc)
{
    Pen pen = sf => sf
        .UseResource(3, ResourceType.Instance.Iron())
        .TakeMark(nameof(TripleStrike), out var layerNum)
        .WriteAttack(11, AttackType.Instance.Physical())
            .WithModify(last => last.Power += layerNum.Value)  // 基础 11 + 叠层数
        .WriteResource(0.5f, ResourceType.Instance.Iron())
        .AddMark(nameof(TripleStrike));  // 叠层 +1（下一次使用时取出）

    return DSL.CreateBy(pen);
}
```

执行时间线：
- 第 1 回合使用：`layerNum.Value = 0`（无标记），攻击力 11，写入 1 个 TripleStrike 标记
- 第 2 回合使用：`layerNum.Value = 1`，攻击力 12，取出后写入 1 个标记
- 第 N 回合使用：攻击力 11 + N-1（如果每回合都用）

### 1.3 典型模式：多技能共享叠层

多个技能 `TakeMark` 同一名称即可共享叠层。Cannon 中 Strike、DoubleStrike、TripleStrike、APShell、CannonBarrel 全部 `TakeMark(nameof(TripleStrike), out var layerNum)`——任一技能都能受益于 TripleStrike 建立的叠层。

### 1.4 典型模式：纹章系统（Lancer）

Lancer 的纹章系统展示 Mark 的最复杂用法——多类型标记 + `ifUndo` 条件性执行 + 批量 `TakeMark`：

```csharp
public static class Keys
{
    public const string SkyStrike = nameof(SkyStrike);
    public const string DragonTooth = nameof(DragonTooth);
    public const string TyrantDestruction = nameof(TyrantDestruction);
    public const string TripleStab = nameof(TripleStab);
}

// 扩展方法：每个 Lancer 攻击技能共享的前置逻辑
public static DSL.AttackFile WriteLancerAttack(
    this DSL.SourceFile af, int power, AttackType.CEValue attackType, 
    float APFactor = 1, [CallerMemberName] string skillName = "")
{
    af.TakeMark(
        new HashSet<string> { Keys.SkyStrike, Keys.DragonTooth, Keys.TyrantDestruction, Keys.TripleStab },
        out var layerNums)
        
        // 条件性：有 DragonTooth 标记才部署护甲
        .WriteDefense(new CommonArmor { Power = 2 },
            ifUndo: () => layerNums.Value[Keys.DragonTooth] <= 0)
        
        // 条件性：有 TyrantDestruction 标记才回复
        .WriteRecovery(2, 
            ifUndo: () => layerNums.Value[Keys.TyrantDestruction] <= 0)
        
        // 条件性：有 TripleStab 标记才追加额外效果
        .LoseMHP(1, 
            ifUndo: () => layerNums.Value[Keys.TripleStab] <= 0)
        .WriteAttack(1, AttackType.Instance.Real(),
            ifUndo: () => layerNums.Value[Keys.TripleStab] <= 0)
        .WriteAttack(1, AttackType.Instance.Real(), delayRounds: 1,
            ifUndo: () => layerNums.Value[Keys.TripleStab] <= 0);
    
    // SkyStrike 标记：增加主力攻击的威力
    return af.WriteAttack(power, attackType, APFactor: APFactor, analyzerKey: skillName)
                .WithModify(last => last.Power += layerNums.Value[Keys.SkyStrike] * 2);
}

// 天击：命中后写入 SkyStrike 标记
private static IDSLSourceFile SkyStrike(ISkillContext sc)
{
    Pen pen = sf => sf
        .UseResource(1, ResourceType.Instance.Iron())
        .WriteLancerAttack(3, AttackType.Instance.Physical())
            .WithCallback(AttackStage.Instance.OnHitArmorFirstTime(), nameof(GetPattern))
            .WithInterupt();
    return DSL.CreateBy(pen);
}

// GetPattern 分析器：攻击命中时写入对应纹章标记
[IsAnalyzer(AnalyzerType.DSL)]
public static void GetPattern(Community player, Community enemy, IAnalyzableData analyzableData)
{
    player.Focus.AddMark(analyzableData.AnalyzerKey);  // 立即写入标记
}
```

## 2. 数据冒险详解 ★

### 2.1 问题根源

`out Lazy<T>` 创建的延迟求值对象，其值仅在 `Intent.Execute` 按序执行到达对应 `Sentence` 时才被填充。DSL 链式调用分两个阶段：

| 阶段 | 发生时机 | 数据状态 |
|---|---|---|
| **声明阶段** | Pen lambda 执行（技能声明时） | `Lazy<T>.Value` 不可用——抛出 `InvalidOperationException` |
| **执行阶段** | `Intent.Execute` 执行（判定管线中） | `Lazy<T>.Value` 可用——对应 Sentence 已执行完毕 |

### 2.2 安全访问规则

`Lazy<T>.Value` 只能在以下上下文中访问：

1. **`WithModify` 的 lambda** — 修辞在对应 WriteAttack 之后执行，前面的 Mark 已结算
2. **`ifUndo` 的 lambda** — 在执行阶段求值
3. **`WriteFree` 的 lambda** — 在执行阶段执行（但要注意顺序——确保 Mark 语句在前）
4. **`WithCallback` 引用的 Analyzer 内部** — 回调在攻击结算过程中触发，Mark 操作已执行

### 2.3 正确与错误示例

**✅ 正确 — WithModify 中访问**：
```csharp
.TakeMark("myMark", out var layerNum)
.WriteAttack(5, AttackType.Instance.Physical())
    .WithModify(last => last.Power += layerNum.Value);
// WithModify 是修辞，插在 WriteAttack 之后执行
// 此时 TakeMark 已完成，layerNum.Value 可用
```

**✅ 正确 — ifUndo 中访问**：
```csharp
.TakeMark(new HashSet<string> { "FireMark" }, out var layerNums)
.WriteAttack(5, AttackType.Instance.Physical(),
    ifUndo: () => layerNums.Value["FireMark"] <= 0);
// ifUndo 在执行阶段求值，此时 TakeMark 已完成
```

**✅ 正确 — 顺序敏感**：
```csharp
.TakeMark("myMark", out var layerNum)
.WriteFree(source =>
{
    int count = layerNum.Value;  // ✅ WriteFree 的 lambda 延迟执行
    source.Focus.Get<Health>().LoseHP(count);
});
```

**❌ 错误 — 声明阶段直接访问**：
```csharp
Pen pen = sf =>
{
    sf.TakeMark("myMark", out var layerNum);
    int count = layerNum.Value;  // ❌ 此时 TakeMark 的 Sentence 尚未执行！
    return sf.WriteAttack(count, AttackType.Instance.Physical());
};
```

**❌ 错误 — 同时 TakeMark 在 WithModify 之后**：
```csharp
.WriteAttack(5, AttackType.Instance.Physical())
    .WithModify(last => last.Power += layerNum.Value)  // ❌ layerNum 在后面才定义！
.TakeMark("myMark", out var layerNum);
// 虽然声明顺序如此，但执行顺序是 TakeMark → WriteAttack → WithModify
// 问题在于 C# 变量作用域——layerNum 在声明点之前不可访问
```

编译器会阻止最后一种情况（变量未定义）。但第一种情况（声明阶段访问）编译通过，运行时才崩溃——这是最常见的错误。

## 3. ifUndo 条件性撤销

### 3.1 机制

所有 Write 方法和便捷方法接受 `Func<bool>? ifUndo` 参数。在执行阶段，若 `ifUndo()` 返回 `true`，该语句被跳过。

### 3.2 配合 Mark 实现条件性技能效果

```csharp
.TakeMark(new HashSet<string> { "FireMark", "IceMark", "LightMark" }, out var layerNums)

// 以下语句仅在拥有对应标记时生效
.WriteAttack(3, AttackType.Instance.Physical(),
    ifUndo: () => layerNums.Value["FireMark"] <= 0)    // 无 FireMark → 跳过

.WriteDefense(new CommonArmor { Power = 2 },
    ifUndo: () => layerNums.Value["IceMark"] <= 0)     // 无 IceMark → 跳过

.WriteRecovery(2,
    ifUndo: () => layerNums.Value["LightMark"] <= 0)   // 无 LightMark → 跳过
```

### 3.3 注意

`ifUndo` 的函数在执行阶段求值，因此可以安全访问 `Lazy<T>.Value`。但要注意：
- `ifUndo` 返回 `true` 仅跳过该条语句，不影响链中其他语句
- 多个 `ifUndo` 语句共享同一个 `layerNums` 引用，数据一致性由执行顺序保证

## 4. 自定义效果系统

### 4.1 效果生命周期

效果通过 `WriteEffect` 写入 `EffectAnalyzableData`，在 `OnApplyingEffect` 阶段被转换为 `EffectEntity` 并存入目标 `Effect` 组件。之后在对应 `EffectType` 的 `TakeEffects` 阶段被触发执行。

```csharp
WriteEffect(
    EffectType.CEValue type,           // 触发阶段：AfterAnalyzableDataWritten / AfterTransport / AfterResult
    EffectTargetType.CEValue targetType, // 作用对象：Self / Enemy
    ClapRoundClock entityClock,        // 效果实体的生命周期
    string analyzerKey,                // 分析器键
    int delayRounds = 0,
    float power = 0,
    Func<bool>? ifUndo = null
)
```

### 4.2 持续伤害效果

```csharp
[IsAnalyzer(AnalyzerType.DSL)]
public static void PoisonEffect(Community player, Community enemy, IAnalyzableData data)
{
    var effect = (EffectAnalyzableData)data;
    enemy.Focus.Get<Health>().LoseHP((int)effect.Power);
}

// 写入：持续 3 回合的敌方中毒效果
Pen pen = sf => sf.WriteEffect(
    EffectType.Instance.AfterAnalyzableDataWritten(),
    EffectTargetType.Instance.Enemy(),
    new ClapRoundClock(remainingRounds: 3),
    analyzerKey: nameof(PoisonEffect),
    power: 2
);
```

### 4.3 Buff 效果

```csharp
[IsAnalyzer(AnalyzerType.DSL)]
public static void RegenBuff(Community player, Community enemy, IAnalyzableData data)
{
    var effect = (EffectAnalyzableData)data;
    player.Focus.Get<Health>().GainHP((int)effect.Power);
}

// 写入：持续 3 回合的己方回复效果
Pen pen = sf => sf.WriteEffect(
    EffectType.Instance.AfterAnalyzableDataWritten(),
    EffectTargetType.Instance.Self(),
    new ClapRoundClock(remainingRounds: 3),
    analyzerKey: nameof(RegenBuff),
    power: 1
);
```

### 4.4 效果内使用 Mark

在 Analyzer 内部可直接使用 Body 扩展方法的 Mark（立即执行版本），因为此时已在执行阶段：

```csharp
[IsAnalyzer(AnalyzerType.DSL)]
public static void StackingEffect(Community player, Community enemy, IAnalyzableData data)
{
    var effect = (EffectAnalyzableData)data;
    int stacks = player.Focus.CountMark("EffectStacks");
    enemy.Focus.Get<Health>().LoseHP((int)effect.Power + stacks);
    player.Focus.AddMark("EffectStacks");
}
```

## 5. 自定义防御系统

### 5.1 DefenseEntity 子类（纯数据声明）

```csharp
public class MyArmor : DefenseEntity
{
    public required string Name { get; init; } = nameof(MyArmor);
    public required string AnalyzerKey { get; init; } = nameof(MyArmorAnalyzer);
    public string MergeKey { get; init; } = nameof(StandardAnalyzers.DefaultMerge);
    public required DefenseType.CEValue Type { get; init; } = MyDefenseType;
    public required int Power { get; set; } = 0;
    public required ClapRoundClock Clock { get; init; } = new(isInfinite: true);
}
```

### 5.2 防御分析器

```csharp
// 签名固定：player, enemy, defense, attackData
[IsAnalyzer(AnalyzerType.Defense)]
public static void MyArmorAnalyzer(Community player, Community enemy,
    DefenseEntity defense, AttackAnalyzableData attackData)
{
    // defense.Power: 当前防御值
    // attackData.Power: 当前攻击力（可能已被前面的防御削减）
    // attackData.TotalDamage: 累积伤害
    
    var absorbed = Math.Min(attackData.Power, defense.Power);
    defense.Power -= absorbed;          // 护甲消耗
    attackData.Power -= absorbed;       // 攻击被削减
    attackData.TotalDamage += absorbed; // 记录累积伤害
}
```

### 5.3 复杂防御示例：反弹 + 自身回复

```csharp
[IsAnalyzer(AnalyzerType.Defense)]
public static void ReactiveArmor(Community player, Community enemy,
    DefenseEntity defense, AttackAnalyzableData attackData)
{
    var absorbed = Math.Min(attackData.Power, defense.Power);
    defense.Power -= absorbed;
    attackData.Power -= absorbed;
    attackData.TotalDamage += absorbed;
    
    // 吸收后反弹等量真实伤害
    DSL.CreateBy(sf => sf
        .WriteAttack(absorbed, AttackType.Instance.Real()))
        .Compile().Execute(player);
}
```

### 5.4 常用 DefenseEntity 声明模板

直接复用内置分析器，只需声明数据类型：

```csharp
// 护甲（值随吸收减少）
new CommonArmor { Name = "...", Power = 5, Clock = new(isInfinite: true) }

// 减伤（值不减少）
new CommonReduction { Name = "...", Power = 3, Clock = new() }

// 荆棘减伤
new ThornReduction { Name = "...", Power = 4, Clock = new() }

// 魔法/物理免疫
new MagicalImmunity { Name = "...", Power = 0, Clock = new() }
new PhysicalImmunity { Name = "...", Power = 0, Clock = new() }

// 百分比减伤
new PercentageReduction { Name = "...", Power = 30, Clock = new() }  // 减 30%
```

## 6. 动态注册高级模式

### 6.1 延迟触发（下回合生效）

```csharp
new ModifierCallback()
{
    AnalyzerKey = nameof(MyDelayedEffect),
    Stage = JudgeStage.Instance.OnBegin(),
    Clock = new(delayRounds: 1),      // 下回合才 IsRinging
    IsPlayer = sc.Self.IsPlayer,
    ModifierOrder = ModifierOrder.Before
}
```

### 6.2 持续多回合

```csharp
new ModifierCallback()
{
    AnalyzerKey = nameof(MySustainedEffect),
    Stage = JudgeStage.Instance.OnEnd(),
    Clock = new(remainingRounds: 3),   // 持续 3 回合
    IsPlayer = sc.Self.IsPlayer,
    ModifierOrder = ModifierOrder.After
}
```


### 6.3 回调内使用 Mark 进行跨回合通信

```csharp
[IsAnalyzer(AnalyzerType.JudgeCallback)]
public static void CounterAttackWatcher(Community player, Community enemy)
{
    // 检查本回合是否受到攻击
    var attacks = player.Focus.Get<TurnContext>().Get<AttackAnalyzableData>();
    if (attacks.Any(a => a.Clock.IsRinging && a.TotalDamage > 0))
    {
        player.Focus.AddMark("WasHit");  // 设置标记供下次技能读取
    }
}
```

## 7. 被动技能

重写 `PassiveSkillImpl(ISkillContext sc)`：

```csharp
public override IDSLSourceFile PassiveSkillImpl(ISkillContext sc)
{
    Pen pen = sf => sf
        .WriteDefense(new RealReduction 
        { 
            Name = "PassiveArmor",
            AnalyzerKey = nameof(StandardAnalyzers.DefaultReduction),
            Type = DefenseType.Instance.RealReduction(),
            Power = 2,
            Clock = new() 
        });
    return DSL.CreateBy(pen);
}
```

被动技能每回合自动执行。`PassiveSkill` 封装方法自动设置 `IsPassive = true`。

## 8. 装备技能与子职业模式

装备技能是一种特殊技能，使用后通常会：
1. 禁用原有技能包
2. 添加新的子职业技能包
3. 部署特殊防御（例如物免 + 高装甲）

```csharp
[IsEquipmentSkill]
private IDSLSourceFile Nightmare(ISkillContext sc)
{
    // 添加子职业包
    sc.Self.Focus.Get<Skill>().AddPackage(new NightmareProfession());
    
    Pen pen = sf => sf
        .UseResource(1, ResourceType.Instance.Dream())
        .LoseHP(5)
        .WriteDefense(new PhysicalImmunity 
        { 
            Name = "Nightmare_Immunity",
            AnalyzerKey = nameof(StandardAnalyzers.PhysicalImmunity),
            Type = DefenseType.Instance.PhysicalImmunity(),
            Power = 0,
            Clock = new(isInfinite: true) 
        })
        .WriteDefense(new NightmareArmor 
        { 
            Name = "Nightmare_Armor",
            AnalyzerKey = nameof(NightmareArmorAnalyzer),
            Type = NightmareArmorType,
            Power = 6,
            Clock = new(isInfinite: true) 
        });

    return DSL.CreateBy(pen);
}
```

子职业的装甲破裂后，通常在防御分析器中恢复入口技能并移除子职业包：

```csharp
[IsAnalyzer(AnalyzerType.Defense)]
public static void NightmareArmorAnalyzer(Community player, Community enemy,
    DefenseEntity defense, AttackAnalyzableData attackData)
{
    var absorbed = Math.Min(attackData.Power, defense.Power);
    defense.Power -= absorbed;
    attackData.Power -= absorbed;
    attackData.TotalDamage += absorbed;
    
    if (defense.Power <= 0)
    {
        // 装甲破裂：恢复入口技能，移除子职业包
        player.Focus.Get<Skill>().AddSkill("梦魇");
        player.Focus.Get<Skill>().RemovePackage(nameof(NightmareProfession));
    }
}
```

## 9. ISudoOperations 越权操作

`ISudoOperations` 提供超越当前回合上下文的操作能力：

```csharp
public interface ISudoOperations
{
    GameInstance DeepCopy(int preRounds = 0);   // 深拷贝当前游戏状态
    bool IsPlayer(Community community);          // 判断阵营
    IGameMetadata GameMetadata { get; }
}
```

`DeepCopy()` 主要用于 AI 模拟（MCTS 搜索），在技能逻辑中较少直接使用。

## 10. 核心模式速查

| 需求 | 方案 |
|---|---|
| 跨回合叠层 | Mark 系统：`AddMark` / `TakeMark` + `out Lazy<T>` + `WithModify` |
| 条件性效果 | `ifUndo` + Mark 计数 |
| 攻击阶段钩子 | `WithCallback(AttackStage, analyzerKey)` |
| 自定义效果 | `[IsAnalyzer(AnalyzerType.DSL)]` + `WriteEffect` |
| 自定义防御 | `DefenseEntity` 子类 + `[IsAnalyzer(AnalyzerType.Defense)]` |
| 改写判定阶段 | `ModifierCallback` / `OverrideCallback` + `RegistCallbackOnJudge` |
| 延迟/持续效果 | `ClapRoundClock(delayRounds/remainingRounds)` |
| 被动技能 | `override PassiveSkillImpl` |
| 子职业切换 | `AddPackage` / `RemovePackage` |
