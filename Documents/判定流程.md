# 判定流程

从一次技能声明出发，追踪 HighPerformanceBlacksmith 判定系统的完整执行过程。

---

## 1. 总览

```
Skill Declare  →  Compile  →  Intent  →  TurnContext (AnalyzableDatas)
                                                   ↓
                                       JudgeRuleManager (9-Stage Chain)
                                                   ↓
                                          Body.GetView() (前端快照)
```

| 阶段 | 做什么 | 核心代码 |
|---|---|---|
| 声明 | `TryDeclare` → `Declare`，产出 `SourceFile` | `Skill`, `GameInstance.Declare` |
| 编译 | `SourceFile.Compile(judgeRuleManager)`，产出 `Intent` | `BlacksmithDSL.SourceFile.Compile` |
| 写入 | `Intent.Execute` 将 AnalysableData 写入 `TurnContext` | `Judger.TranslateIntentsToResolutions` |
| 阶段链 | 9 个 `JudgeStage` 按优先级依次执行 | `JudgeRuleManager.Judge` |
| 执行 | 各阶段内 AnalysableData 通过 `AnalyzerRegistry` 调度结算 | `TurnContext.ExecuteImpl`, `AnalyzerRegistry.DSL.Get(key)(...)` |
| 清理 | 组件 Update、过期规则移除、Community 更新 | `OnUpdating`, `OnEnd` |

---

## 2. 声明：从输入到 SourceFile

`GameInstance.Declare` 是每回合的入口：

```csharp
// GameInstance.cs
Metadata.UpdateCurrentSkill(skillName, esn);  // 更新本回合双方技能元数据快照

var playerContext = new DefaultSkillContext(this, skillName, Player, param, stringParam);
var enemyContext = new DefaultSkillContext(this, esn, Enemy, ep, esp);

var ps = Player.Focus.Get<Skill>();
var psfs = ps.GetPassiveSkill(playerContext);  // IEnumerable<IDSLSourceFile>
foreach (var s in Player.SummonList)
    psfs = psfs.Concat(s.Get<Skill>().GetPassiveSkill(playerContext));
psfs = psfs.Append(ps.Declare(skillName, playerContext));

var es = Enemy.Focus.Get<Skill>();
var esfs = es.GetPassiveSkill(enemyContext);
foreach (var s in Enemy.SummonList)
    esfs = esfs.Concat(s.Get<Skill>().GetPassiveSkill(enemyContext));
esfs = esfs.Append(es.Declare(esn, enemyContext));

Judger.Judge(psfs, esfs);
History.SkillHistory.Add((playerContext, enemyContext));
```

`ISkillContext` 包含 `SudoOperations`、`SkillName`、`Self`（己方 `Community`）、`Param`、`StringParam`、`SkillHistory`、`GameMetadata`。

## 3. 编译：SourceFile → Intent

`SourceFile.Compile(judgeRuleManager)` 分三步：

1. **展开修辞缓存**：弹出 `WithCallback`/`WithModify`/`WithBloodSuck`/`WithInterupt` 等修辞 Sentence，插入到其绑定的主语句之后
2. **合并句子**：合并所有 `Sentence.Structure` 为单个 `Action<Community>`（多播委托）
3. **注册动态回调**：如果 `judgeRuleManager != null`，在合并后的委托末尾追加 `_callbacksOnCompile` 的落地逻辑

```csharp
// BlacksmithDSL.Skill.cs
public Intent Compile(JudgeRuleManager? judgeRuleManager = null)
{
    List<Sentence> sentences = new(_sentences);
    int n = _rhetoricCache.Count;
    for (int i = 0; i < n; ++i)
    {
        var rhetoric = _rhetoricCache.Pop();
        int index = sentences.IndexOf(rhetoric.BindSentence!) + 1;
        if (index > 0)
        {
            sentences.Insert(index, new(rhetoric.Structure, rhetoric.SentenceType, rhetoric.StructureType));
        }
    }
    Action<Community> result = (a) => { };
    foreach (var sentence in sentences)
    {
        result += sentence.Structure;
    }
    if (judgeRuleManager != null)
    {
        result += _ =>
        {
            foreach (var callback in _callbacksOnCompile)
            {
                judgeRuleManager.AddJudgeRule(callback);
            }
        };
    }
    return new Intent() { Execute = result };
}
```

`Intent` 只有一个 `Execute` 委托——执行它就将技能的全部效果写入 `TurnContext`。

## 4. 写入：Intent → TurnContext

### 4.1 TranslateIntentsToResolutions

```csharp
foreach (var temp in _playerIntents)
{
    temp.Execute(Player);
}
foreach (var temp in _enemyIntents)
{
    temp.Execute(Enemy);
}
```

每个 `Intent.Execute` 内部是合并后的 `Sentence.Structure` 链。以 `WriteAttack` 为例——Sentence 中构造 `AttackAnalyzableData`，设置 `AnalyzerKey`、`Clock`、`Type`、`Power` 等纯数据字段，然后调用 `source.Focus.Get<TurnContext>().WriteAnalyzableData(analyzableData)`。

### 4.2 预处理钩子

```csharp
// TurnContext.cs
public override void WriteAnalyzableData(IAnalyzableData analyzableData)
{
    var pp = _preprocesses[analyzableData.GetType()];
    pp.ForEach(p => 
    { 
        if (p.Clock.IsRinging) 
        {
            p.Action(analyzableData);
        }
    });
    base.WriteAnalyzableData(analyzableData);
}
```

`AddPreprocess<TAnalyzableData>(Action<TAnalyzableData>, delayRounds, remainingRounds, isInfinite)` 在写入前拦截数据。每个预处理钩子携带独立的 `ClapRoundClock`。

### 4.3 四种 AnalysableData

| 数据类型 | 写入语句 | 关键字段 |
|---|---|---|
| `AttackAnalyzableData` | `WriteAttack` | `AnalyzerKey`, `Type` (AttackType), `Power` (int), `Clock`, `APFactor`, `TotalDamage`, `StageKeys` |
| `DefenseAnalyzableData` | `WriteDefense` | `AnalyzerKey`, `Defense` (DefenseEntity), `Clock` |
| `EffectAnalyzableData` | `WriteEffect` | `AnalyzerKey`, `Type` (EffectType), `TargetType`, `Power`, `Clock`, `EntityClock` |
| `ResourceAnalyzableData` | `WriteResource` | `AnalyzerKey`, `Type` (ResourceType), `Power` (float), `Clock` |

`Clock.IsRinging`（`DelayRounds == 0 && RemainingRounds > 0 && !forceKill()`）的在本回合结算，否则调用 `Clock.RoundPass()` 推进并顺延。

## 5. 阶段链：9 个 JudgeStage

### 5.1 StageRuleContainer

```csharp
// JudgeRuleManager.cs
public class RuleUnit : IAnalyzableData
{
    public required string AnalyzerKey { get; init; }
    public required ClapRoundClock Clock { get; init; }
    public required bool IsPlayer { get; init; }
}
```

```csharp
class StageRuleContainer
{
    Action<Community, Community> _baseRule;
    List<RuleUnit> _overrideRules;
    List<RuleUnit> _modifiersBefore;
    List<RuleUnit> _modifiersAfter;
}
```

执行顺序：**Before Modifiers → 核心（Override 或 Base） → After Modifiers**。全通过 `AnalyzerRegistry.JudgeCallback` 调度。每阶段执行完毕后自动 `Update()`：移除 `IsDead` 规则，并对剩余规则调用 `Clock.RoundPass()`。

```csharp
// StageRuleContainer.Execute 核心逻辑
// Before modifiers
foreach (var rule in _modifiersBefore)
{
    if (rule.Clock.IsRinging)
    {
        if (rule.IsPlayer)
            AnalyzerRegistry.JudgeCallback.Get(rule.AnalyzerKey)(player, enemy);
        else
            AnalyzerRegistry.JudgeCallback.Get(rule.AnalyzerKey)(enemy, player);
    }
}
// 核心规则（Override 或 Base）
if (overrideRule == null)
    _baseRule(player, enemy);
else
    AnalyzerRegistry.JudgeCallback.Get(overrideRule.AnalyzerKey)(
        overrideRule.IsPlayer ? player : enemy,
        overrideRule.IsPlayer ? enemy : player);
// After modifiers（同理）
```

### 5.2 阶段一览

| 优先级 | 阶段 | 默认规则 | 说明 |
|---|---|---|---|
| 0 | `OnBegin` | 空 | 留给动态注册——状态检查、Buff 重置 |
| 8 | `OnApplyingEffect` | `ApplyEffect` | 执行 EffectAnalyzableData，创建 EffectEntity 并存入目标 Effect 组件 |
| 16 | `OnEffectTaking_AfterAnalyzableDataWritten` | `TakeEffects(AfterAnalyzableDataWritten)` | 遍历双方 Focus 的 Effect 组件，执行对应类型的 EffectEntity |
| 32 | `OnAttackCanceling` | `CancelAttacks` | 双方攻击按 Type 排序后 1:1 对消，跳过真实伤害和延迟攻击 |
| 64 | `OnEffectTaking_AfterTransport` | `TakeEffects(AfterTransport)` | 执行传输后生效的效果 |
| 128 | `OnApplyingOthers` | `ApplyOthers` | 依次：防御部署 → 攻击结算（装甲穿透 + HP 扣除） → 资源获取。最密集的阶段 |
| 256 | `OnUpdating` | `Update` | `Community.Update()`——清理召唤物、推进 Transform 时钟、更新所有 Body 组件 |
| 1024 | `OnEffectTaking_AfterResult` | `TakeEffects(AfterResult)` | 结算完成后的效果触发 |
| 2048 | `OnEnd` | 空 | 留给动态注册——回合结束时的清理和重置 |

### 5.3 攻击结算

攻击在 `OnApplyingOthers` 中由 `TurnContext.ExecuteImpl` 取出（只处理 `Clock.IsRinging` 的，执行后移除；否则调用 `Clock.RoundPass()` 推进）。执行通过 `AnalyzerRegistry.DSL.Get(analyzableData.AnalyzerKey)` 调度到 `StandardAnalyzers.DefaultAttack`：

```
真实伤害？
  ├─ 是 → 跳过防御，直接命中
  └─ 否 → 遍历防御列表（按优先级）：
            ├─ 首次命中 armorList → 触发 OnHitArmorFirstTime 阶段回调
            ├─ APList 中的防御 → Power *= APFactor（穿甲）
            ├─ AnalyzerRegistry.Defense.Get(defense.AnalyzerKey)(enemy, player, defense, attackData) → 防御计算
            ├─ APList 中的防御 → Power /= APFactor（还原）
            └─ Power <= 0 → 触发 OnEnd 回调，结束

触发 OnHitBody 回调（WithCallback 注册的 OnHitBody 分析器）
→ LoseHP → TotalDamage += Power → 触发 OnEnd 回调
```

APList：`ThornReduction`, `CommonReduction`, `StoneShell`, `RealArmor`, `CommonArmor`
ArmorList：`StoneShell`, `RealArmor`, `CommonArmor`

### 5.4 其他数据执行

- **DefenseAnalyzableData**：`AnalyzerRegistry.DSL.Get(key)(player, enemy, data)` → `StandardAnalyzers.DefaultDefense` → `Defense.Add(defenseData.Defense)`。同类型且 `CanMerge` 则合并，否则追加。列表按优先级排序。
- **EffectAnalyzableData**：`ApplyEffect` 阶段创建 `EffectEntity(type, power, entityClock)`，设置 `AnalyzerKey`，存入目标 `Effect` 组件。后续 `TakeEffects` 阶段按 `EffectType` 筛选，通过 `AnalyzerRegistry.DSL.Get(entity.AnalyzerKey)(...)` 执行。
- **ResourceAnalyzableData**：`AnalyzerRegistry.DSL.Get(key)(player, enemy, data)` → `StandardAnalyzers.DefaultResource` → `Resource.Gain(type, power)`。

## 6. 动态注册

### 6.1 核心类型

| 类型 | 说明 |
|---|---|
| `ICallbackOnJudge` | 接口，定义 `AnalyzerKey`（string）、`Clock`（`ClapRoundClock`）、`Stage`（`JudgeStage.CEValue`）、`IsPlayer`（bool） |
| `ModifierCallback` | 实现 `ICallbackOnJudge`，额外携带 `ModifierOrder`（`Before` / `After`） |
| `OverrideCallback` | 实现 `ICallbackOnJudge`，直接替换阶段的默认规则 |

### 6.2 注册链路

技能 DSL 中 `RegistCallbackOnJudge(callbacks)` → 记录到 `_callbacksOnCompile`。`Compile(judgeRuleManager)` 时在委托末尾追加注册逻辑：

```csharp
// Compile 内部：RegisterCallbacks
result += _ =>
{
    foreach (var callback in _callbacksOnCompile)
    {
        judgeRuleManager.AddJudgeRule(callback);
    }
};
```

```csharp
// JudgeRuleManager.cs
public void AddJudgeRule(List<ICallbackOnJudge> callbacks)
{
    foreach (var callback in callbacks)
    {
        RuleUnit unit = new()
        {
            AnalyzerKey = callback.AnalyzerKey,
            Clock = callback.Clock,
            IsPlayer = callback.IsPlayer
        };
        if (callback is OverrideCallback)
            _ruleContainers[callback.Stage].AddOverride(unit);
        else if (callback is ModifierCallback modifierCallback)
            _ruleContainers[callback.Stage].AddModifier(unit, modifierCallback.ModifierOrder);
    }
}
```

回调方法必须标记 `[IsAnalyzer(AnalyzerType.JudgeCallback)]`：
```csharp
[IsAnalyzer(AnalyzerType.JudgeCallback)]
public static void MyCallback(Community player, Community enemy) { ... }
```

### 6.3 模板专门化

`ICallbackOnJudge` 通过 `IsPlayer` 标志固化视角。注册时 `RegistCallbackOnJudge` 内部根据 `source.IsPlayer` 设置每个 callback 的 `IsPlayer`。

执行时：
- `IsPlayer == true` → `AnalyzerRegistry.JudgeCallback.Get(key)(player, enemy)`
- `IsPlayer == false` → `AnalyzerRegistry.JudgeCallback.Get(key)(enemy, player)`

回调方法始终接收 `(player, enemy)` 二参数，框架自动处理双方身份的对应关系。

### 6.4 单链执行

```csharp
// JudgeRuleManager.cs
public override void Judge(Community player, Community enemy)
{
    foreach (var stage in _ruleContainers)
    {
        stage.Value.Execute(player, enemy);
    }
}
```

简化单链：每个阶段的 `StageRuleContainer.Execute()` 内部依次执行 Before Modifiers → 核心规则 → After Modifiers，全部通过 `AnalyzerRegistry.JudgeCallback` 调度，执行完毕后自动 `Update()`。

## 7. 清理与更新

| 阶段 | 行为 |
|---|---|
| `OnUpdating` | `Community.Update()`——清理已死亡召唤物并触发回调、推进 Transform 时钟、更新 Focus 和各 Summon Body 的组件（Health/Defense/Effect/Skill）、TurnContext.Update() 清理过期预处理 |
| `OnEnd` | 空——留给动态注册做回合结束时的清理和重置 |

## 9. ClapRoundClock 时钟机制

所有 AnalysableData、预处理钩子、动态规则（RuleUnit）、EffectEntity 和 Community Transform 统一使用 `ClapRoundClock` 管理时序：

| 属性/方法 | 说明 |
|---|---|
| `DelayRounds` | 延迟回合数，>0 时 `RoundPass()` 优先递减此项 |
| `RemainingRounds` | 剩余生效回合数，DelayRounds 归零后才递减 |
| `IsRinging` | `DelayRounds == 0 && RemainingRounds > 0 && !forceKill()`——当前回合应当执行 |
| `IsDead` | `RemainingRounds <= 0 \|\| forceKill()`——已失效，应被清理 |
| `RoundPass()` | 推进一回合 |
| `isInfinite` | 是否无限持续（不递减 RemainingRounds） |
| `forceKill` | `Func<bool>`，返回 true 时强制标记为失效 |
